# Plan: Equity Engine API (Phase 2)

## Summary
Expose a Rust function that computes pre-flop hand-vs-hand equity using the existing `pokers` crate, surface it through `wasm-bindgen` as `equity_vs(hand_a, hand_b) -> f32`, and prove correctness with Rust property tests against `pokers` directly. Wire it into the JS engine wrapper as `EngineApi.equityVs(...)`. No exercise UI yet — this phase delivers a trustworthy number.

## User Story
As the project owner, I want a correctly computed pre-flop hand-vs-hand equity number callable from the browser, so that the eventual exercise screen has a single source of truth for every "what's the equity?" prompt.

## Problem → Solution
Current state: `engine` exposes only `count_combos`. `pokers` has the equity calculator (`exact_equity`) but it's not wired through wasm-bindgen, has no Rust-side tests in this crate, and no JS surface. The frontend's `lib/engine.ts` is single-method.

Desired state: A Rust function `equity_vs(hand_a: &str, hand_b: &str) -> Result<f32, JsError>` that builds two single-combo `HandRange`s, calls `exact_equity` with empty board/dead masks, and returns the first range's equity as `f32`. Property tests cross-check the wrapper against direct `pokers` calls and verify `equity(A,B) + equity(B,A) ≈ 1.0`. The JS wrapper exposes `equityVs(handA, handB): number`, with a Vitest spec covering canonical poker spots. The home page gets a tiny additional smoke output: typing a second hand returns the equity number — confirms the WASM round-trip end-to-end on Pixel.

## Patterns to Mirror

### RUST_WASM_EXPORT
```rust
// SOURCE: engine/src/lib.rs:4-8
#[wasm_bindgen]
pub fn count_combos(range_str: &str) -> u32 {
    let ranges = HandRange::from_strings(vec![range_str.to_string()]);
    ranges[0].hands.len() as u32
}
```
Top-level `#[wasm_bindgen]` on a free function. snake_case Rust → snake_case JS export. Borrow `&str` as input — wasm-bindgen handles the JS-string crossing.

### RUST_TEST_STRUCTURE
```rust
// SOURCE: engine/src/lib.rs:10-23
#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn pocket_aces_has_six_combos() {
        assert_eq!(count_combos("AA"), 6);
    }
}
```
Inline `#[cfg(test)] mod tests` per file, `snake_case` test names that read as a sentence. New tests go in the same module.

### TS_ENGINE_API_INTERFACE
```ts
// SOURCE: web/src/lib/engine.ts:3-5
export interface EngineApi {
  countCombos(rangeStr: string): number;
}
```
PascalCase interface, camelCase methods (`countCombos` wraps `count_combos`). Add new methods to this interface; do not introduce a second one.

### TS_LAZY_SINGLETON
```ts
// SOURCE: web/src/lib/engine.ts:7-23
let cachedPromise: Promise<EngineApi> | null = null;

export function getEngine(): Promise<EngineApi> {
  if (cachedPromise === null) {
    cachedPromise = init()
      .then<EngineApi>(() => ({
        countCombos(rangeStr: string): number {
          return count_combos(rangeStr);
        },
      }))
      .catch((err: unknown) => {
        cachedPromise = null;
        throw err;
      });
  }
  return cachedPromise;
}
```
Promise cached at module scope; cleared on rejection so a retry is possible. New methods extend the resolved object — no second initializer.

### CANCELLED_EFFECT

// SOURCE: web/src/app/page.tsx:11-26

Async effect uses a `cancelled` flag. Mirror exactly for any new effect.

## Step-by-Step Tasks

### Task 1: Add `equity_vs` to `engine/src/lib.rs`
- **ACTION**: Add a new `#[wasm_bindgen]` function that computes pre-flop hand-vs-hand equity.
- **IMPLEMENT**:
  - Signature: `pub fn equity_vs(hand_a: &str, hand_b: &str) -> Result<f32, JsError>`.
  - Build ranges: `HandRange::from_strings(vec![hand_a.to_string(), hand_b.to_string()])`.
  - Validate both ranges have at least one combo each — if either `ranges[i].hands.is_empty()`, return `Err(JsError::new("invalid hand"))`.
  - Call `exact_equity(&ranges, 0u64, 0u64, 1, Arc::new(AtomicBool::new(false)), |_progress: u8| {})`.
  - Map `SimulatorError::ConflictingRanges` → `JsError::new("hands conflict")`. Other simulator errors are unreachable for our 2-player no-board call but still map them to a generic `JsError::new(format!("equity failed: {e:?}"))` rather than `unwrap()`.
  - Return `result.equities[0] as f32`.
- **MIRROR**: `RUST_WASM_EXPORT` for the function shape.
- **IMPORTS**:
  - `use pokers::{HandRange, exact_equity};`
  - `use std::sync::Arc;`
  - `use std::sync::atomic::AtomicBool;`
  - `use wasm_bindgen::JsError;` (already importing `wasm_bindgen::prelude::*`, but `JsError` is in the prelude in v0.2)
- **GOTCHA**:
  - `pokers` requires ≥ 2 ranges (`MIN_PLAYERS = 2`); we always pass exactly 2 — fine.
  - `HandRange::from_string` lowercases the input internally, so `"AsKs"` and `"asks"` both work. The JS wrapper need not normalize.
  - The `web` feature flag (`features = ["web"]` in `Cargo.toml`) makes `exact_equity` ignore `n_threads` and run on the calling thread. Pass `1` to keep intent clear.
  - `f64 → f32` conversion is lossy in theory (52→23 mantissa bits). Equity is always in `[0.0, 1.0]`; f32 has ~7 decimal digits of precision there, well within the 0.5% PRD tolerance. Do the cast at the boundary, not inside `pokers`.
  - Do NOT use `unwrap()` on the `exact_equity` result. The PRD success metric is correctness, but a panic crashing the WASM module is a much worse user experience than a thrown JS error.
  - The `callback` parameter must accept `u8` and be `Send + Sync`. A closure `|_: u8| {}` satisfies this. Do NOT pass a `Box<dyn Fn>`.
- **VALIDATE**:
  - `cd engine && cargo check` — compiles
  - `cd engine && cargo clippy -- -D warnings` — no warnings

### Task 2: Add Rust property tests for `equity_vs`
- **ACTION**: Extend `#[cfg(test)] mod tests` with tests that lock in correctness against `pokers` direct calls and against poker-known truths.
- **IMPLEMENT**:
  - **Spot tests** (deterministic, fast):
    - `aces_vs_kings_equity_around_81_percent`: `equity_vs("AsAh", "KsKh")` is in `(0.80, 0.83)`.
    - `cooler_setup_aks_vs_qq_equity_around_43_percent`: `equity_vs("AsKs", "QhQd")` is in `(0.42, 0.44)`. Matches the PRD success signal.
    - `mirror_pair_aces_equity_is_half`: `equity_vs("AsAh", "AdAc")` is in `(0.49, 0.51)`.
    - `invalid_hand_returns_error`: `equity_vs("garbage", "AsKs").is_err()`.
    - `conflicting_hands_return_error`: `equity_vs("AsKs", "AsQh").is_err()` (both want `As`).
  - **Property test (`proptest`)**:
    - Generate two random non-overlapping hold'em hands (4 distinct cards, sampled from 0..52 with `rank*4+suit`).
    - Format as 4-char strings (`As`, `Kh`, etc.).
    - Assert: `eq_ab + eq_ba` is within `1e-3` of `1.0` (catches input-ordering bugs and anything that breaks the "all outcomes accounted for" invariant — ties split evenly into the equity by `pokers`).
    - Cap to `cases: 16` via `proptest_config!` — exact enumeration is ~500ms each on CI.
- **MIRROR**: `RUST_TEST_STRUCTURE`.
- **IMPORTS**: at the top of the test module:
  - `use proptest::prelude::*;`
- **GOTCHA**:
  - Sampling 4 distinct cards: use `proptest::sample::subsequence((0u8..52).collect::<Vec<_>>(), 4)` or generate `(u8, u8, u8, u8)` and reject duplicates with `prop_assume!`. The latter is simpler.
  - Card → string: `rank` index 0..13 maps to `"23456789TJQKA"`, `suit` 0..4 to `"shdc"` (verify against `pokers/src/constants.rs` if in doubt — but the property only requires round-trip consistency, not absolute card identity).
  - The `eq_ab + eq_ba ≈ 1` invariant holds because `exact_equity` sums wins + halved ties to 1 across players in any heads-up pre-flop spot with no board/dead cards. Document this in a comment so the test isn't mistaken for a tautology.
  - `cargo test` under `feature = "web"` may or may not be the default — the engine's only consumer is wasm-pack which sets the feature explicitly. Run `cargo test` first; if the threaded path is needed and missing, run `cargo test --features web`. Do NOT add `default = ["web"]` unprompted.
- **VALIDATE**: `cd engine && cargo test` — all spot tests pass, the property test runs `cases: 16` to completion within ~10s. No regression on the 2 existing `count_combos` tests.

## Risks
| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| WASM bundle bloats past 80 KB gzipped after pulling in `exact_equity` | MEDIUM | Medium (PRD metric) | `pokers` is already a dep; the equity calculator may pull in more lookup tables. | Vitest can't load the bundler-target wasm in Node (worker-loader / `import.meta.url` quirks) | LOW | Medium | Fallback to a `--target nodejs` build under `engine/pkg-node/` consumed only by Vitest via a path alias. Add a follow-up if encountered, do not preemptively split |
| `exact_equity` is slower than expected on Pixel and the smoke page feels laggy | MEDIUM | Low | Accept it for Phase 2; the smoke page is throwaway. Phase 4 caches per problem and considers `approx_equity` if needed |
| `f64 → f32` cast loses precision below the 0.5% PRD tolerance | LOW | Low | Equity is in `[0,1]`; f32 has > 6 decimal digits of precision there. PRD tolerance is 0.5% (1 part in 200) — orders of magnitude looser than f32 resolution |
